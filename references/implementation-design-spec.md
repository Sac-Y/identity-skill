# 构建前设计说明（Pre-Build Design Note）

在参考图和关键素材确认后、写代码前使用本说明。

目标是锁定少量能够防止视觉漂移的关键决策。保持简短；它是交接说明，不是完整实施规格。

## 输出格式

```text
构建前设计说明

主要基线：
- [哪张参考图或哪个 section 定义全站视觉系统]

Section 顺序：
- [按顺序列出 section]

全局外框与排版：
- Nav/header：[logo/wordmark、导航位置、active state、桌面/移动端行为]
- 字体锁定：[必须保持一致的 display、body、label、nav 角色]
- 配色/材质：[共享背景、强调色、图像处理、纹理]

必须保留的锚点：
- [实现后仍必须存在的 5-8 个具体视觉锚点]

动效与交互：
- [只写必要的 hover、scroll、transition 或媒体行为]
- Reduced motion：[需要时写降级方案]

素材与来源决策：
- 生成素材：[使用位置]
- 官方/有来源素材：[必须使用的位置]
- 代码渲染的标记/文字：[logo、nav、button、label]
- 未解决阻塞：[阻止高保真复刻的素材质量或参考图冲突]

允许偏差：
- [移动端简化、弱参考图决策或明确批准的变化]

交接：
- 以已确认参考图为设计目标。
- `frontend-app-builder` 可用时，由它负责详细实现和保真 QA。
```

## 规则

- 不要把它扩写成长篇设计系统文档。
- 不要重复 `frontend-app-builder` 的实现清单或 QA ledger。
- 不要借此偏离已确认参考图重新设计。
- 不要忽略参考图之间的冲突；写明裁决，或请求重新生成。
- 不要让生成图片承载品牌 logo、官方标记、产品名称、UI label 或导航文字。使用代码渲染文字/标记，或有官方来源的素材。
- 不要把动效当成通用润色。只记录用户要求或参考图明确暗示的动效。
